<Callout type="warn">
  You must have Kotlin Coroutines installed.
</Callout>

With the project being written in Kotlin, Kotlin DSL is heavily supported by default. However, a separate module is available if you want even more extensive Kotlin DSL support **with [Kotlin Coroutines](https://kotlinlang.org/docs/coroutines-overview.html)**.

## Accessing the API

[![Latest Version](https://img.shields.io/badge/dynamic/xml?label=Latest%20Version&style=for-the-badge&logo=gradle&url=https%3A%2F%2Frepo.undefinedcreations.com%2Freleases%2Fcom%2Fundefined%2Fstellar%2Fmaven-metadata.xml&query=%2F%2Fmetadata%2F%2Fversioning%2F%2Flatest)](https://github.com/UndefinedCreations/Stellar/releases)

To use the Kotlin module, you need to add it as a dependency:

```groovy tab="Gradle Groovy DSL"
compile 'com.undefined:stellar-kotlin:(latest version):platform' // or implementation if shadowing
```
```groovy tab="Gradle Kotlin DSL"
compileOnly("com.undefined:stellar-kotlin:(latest version):(platform)") // or implementation if shadowing
```
```xml tab="Maven"
<dependency>
    <groupId>com.undefined</groupId>
    <artifactId>stellar-kotlin</artifactId>
    <version>(latest version)</version>
    <classifier>(platform)</classifier>
    <scope>provided</scope> // should be removed if shadowing
</dependency>
```

<Callout title="If you don't use the plugin" type="warn">
  you need modify your Gradle dependency to `implementation` instead of `compile`/`compileOnly` or remove the `scope` block from your Maven dependency.
</Callout>

## Setup

To get started, make sure to change the scope in `StellarConfig`:

```kt
class MyPlugin : JavaPlugin() {
  val scope = // ...

  override fun onEnable() {
    StellarConfig.setScope(scope)
  }
}
```
